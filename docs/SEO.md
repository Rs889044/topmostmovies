# SEO — topmostmovies.com

SEO is the product. Bake these into shared components so every page is consistent.

## Shared `<Seo>` component

`src/components/Seo.astro` sets all per-page metadata in one place. Every page renders it
in `<head>` (usually via the layout). Proposed props:

```ts
interface SeoProps {
  title: string;             // page <title> (kept under ~60 chars)
  description: string;       // meta description (~150–160 chars)
  canonical?: string;        // defaults to current page URL built from site + path
  ogType?: 'website' | 'article' | 'video.movie';
  image?: string;            // OG/Twitter image (poster/backdrop or site default)
  noindex?: boolean;         // e.g. thin/utility pages
  jsonLd?: object | object[];// structured data block(s) for this page
}
```

It outputs: `<title>`, `<meta name="description">`, **canonical** `<link>`, Open Graph
(`og:title/description/type/image/url/site_name`), Twitter Card
(`summary_large_image`), and `<script type="application/ld+json">` for each `jsonLd` entry.

Usage is documented inline in the component and below.

## JSON-LD structured data (per page type)

Builders live in `src/lib/jsonld.ts`.

| Page | Schema |
|------|--------|
| Homepage | `WebSite` (+ Sitelinks SearchBox if search added) **and** `Organization` |
| List page ("Top 10 …") | `ItemList` (ordered `ListItem`s → movie URLs) + `BreadcrumbList` |
| List page with FAQ | also `FAQPage` |
| Movie page | `Movie` (name, image, datePublished, aggregateRating if available, contentRating) + `BreadcrumbList` |
| Any interior page | `BreadcrumbList` |

- `aggregateRating` only when a real rating exists; `contentRating` only when a real
  certification exists — never emit fabricated values (mirrors the honesty rule).

## Per-page SEO checklist

- [ ] Exactly one `<h1>`; headings in logical order (no skipped levels).
- [ ] Semantic HTML (`<main>`, `<nav>`, `<article>`, `<section>`, `<footer>`).
- [ ] `<title>` unique + keyword-aligned (from [KEYWORDS.md](KEYWORDS.md)).
- [ ] Meta description unique, compelling, ~150–160 chars.
- [ ] Canonical URL set (absolute, https, no trailing inconsistencies).
- [ ] Open Graph + Twitter Card tags (image present).
- [ ] Correct JSON-LD for the page type, validated (no errors).
- [ ] Breadcrumbs rendered (interior pages) + `BreadcrumbList` JSON-LD.
- [ ] Internal links: related lists + (movie pages) "Featured in".
- [ ] Images via Astro `<Image>`: correct sizes, lazy-loaded below the fold,
      descriptive `alt` (e.g. "Parasite (2019) movie poster").
- [ ] Original copy present (intro / blurbs / synopsis) — no thin/duplicate content.
- [ ] FAQ block on key category pages, with `FAQPage` schema.
- [ ] Mobile-first, no layout shift; ad slots reserve space.

## Site-wide SEO assets

- **`sitemap.xml`** — auto-generated by `@astrojs/sitemap` (requires `site` in
  `astro.config.mjs`).
- **`robots.txt`** (`public/robots.txt`) — allow crawling; link to the sitemap.
- **Custom `404`** — `src/pages/404.astro`, with helpful navigation back into the site.
- **Canonical strategy** — every page self-canonical; combo/duplicate-risk pages handled
  deliberately. On Cloudflare, `*.pages.dev` preview gets `noindex` via `public/_headers`
  (Phase 8) so only the `.com` is indexed.

## Internal linking strategy

- List pages link to each movie (cards) and to **related lists** (same dimension siblings,
  cross-dimension where relevant).
- Movie pages link back to **every** list they're featured in ("Featured in" section).
- Homepage links to top entry points for each dimension.
- This dense, tag-driven internal linking is a primary ranking lever for long-tail combos.

## Validation (`npm run validate-seo`)

After `npm run build`, run **`npm run validate-seo`** (`scripts/validate-seo.ts`). It crawls
`dist/` and fails (non-zero exit) on any of:

- not exactly one `<title>` or one `<h1>` per page (404 exempt),
- missing meta description / canonical / `og:title` / `twitter:card`,
- JSON-LD that doesn't parse or lacks `@context`/`@type`,
- wrong structured-data types per page type (home → `WebSite`+`Organization`; list →
  `ItemList`+`BreadcrumbList`; movie → `Movie`+`BreadcrumbList`).

This is our standing structured-data check (complements Google's Rich Results Test, which
can be run manually on the deployed URLs in Phase 8). **Status:** all pages pass.

### Honesty gating in JSON-LD (verified)

`Movie` nodes emit `aggregateRating` / `contentRating` **only when real values exist**, and
**never** emit a `description` built from TMDb's overview — `description` is populated only
from our original `synopsis` (Phase 5). Confirmed: zero overview leaks across movie pages.

## Sitemap & noindex (single source of truth)

`src/lib/seo.ts` defines `NOINDEX_PATHS` (`/about`, `/contact`, `/privacy-policy` — thin
placeholders until Phase 6). The page `<Seo noindex>` flag and the `@astrojs/sitemap`
`filter` both derive from it, so the sitemap lists only indexable pages (66 URLs currently;
404 is auto-excluded by the integration).

## Internal linking (implemented)

- List pages render a **"Explore related lists"** block (`RelatedLists.astro`,
  `relatedLists()` in `src/lib/lists.ts`): sibling lists in the same dimension plus
  cross-dimension lists derived from the list's own movies.
- Movie pages render **"Featured in"** linking every list the movie belongs to.
- Homepage + header link to real populated lists. Net: dense, tag-driven internal linking.

## Core Web Vitals

- Static HTML, minimal JS (Astro islands only where needed; consent banner is the main

- Reserve dimensions for posters and ad slots to keep **CLS ≈ 0**.
- Optimized, sized, lazy images; small Tailwind output (purged). Target LCP via
  prioritized above-the-fold poster.
